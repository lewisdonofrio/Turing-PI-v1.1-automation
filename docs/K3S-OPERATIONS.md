# k3s Operations Guide
# Managing the Lightweight Kubernetes Cluster
# =============================================================================

This document describes operational procedures for managing the k3s cluster on
the Turing PI v1.1 system.

===============================================================================
1. Entering k3s Mode
===============================================================================

To return the cluster to normal k3s operation:

    ansible-playbook -i inventory/hosts site.yml -t k3s_mode

This:
- stops distccd
- unmounts tmpfs
- restores RAM to workloads

===============================================================================
2. Checking Node Status
===============================================================================

On kubenode1:

    kubectl get nodes
    kubectl get pods -A

===============================================================================
3. Restarting k3s
===============================================================================

On any node:

    sudo systemctl restart k3s

===============================================================================
4. Viewing Logs
===============================================================================

    sudo journalctl -u k3s -f

===============================================================================
5. Upgrading k3s
===============================================================================

1. Drain node
2. Stop k3s
3. Replace binary
4. Start k3s
5. Uncordon node

===============================================================================
6. Summary
===============================================================================

This guide provides essential operational commands for managing k3s.
